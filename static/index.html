Operator Guide

Welcome to the CyberForge Control Deck. Here’s how to operate the lab provisioning system.

Deploying a Challenge

  1. Select an Operator ID: Your unique identifier. This groups your sessions together.
  2. Filter the Catalog: Use the type, domain, or difficulty filters to find relevant labs.
  3. Pick a Challenge: Select your desired challenge from the dropdown.
  4. Click Deploy: The system will provision your dedicated environment. Wait for the state to turn ACTIVE.

Managing Active Sessions

  • Reset Lab: If your environment becomes unstable, hit the Reset button to rebuild it from scratch.
  • IP Access: Note the Target and Attacker IPs — you'll need these to interact with the challenge.
  • Concurrency: Only one active mock-lab per user is supported at a time the real VirtualBox provisioner supports multiples depending on infrastructure limits.

Troubleshooting Deployments

If your deploy gets stuck or shows FAILED, check the Audit Explorer tab below for the specific error logged by the orchestration backend. Usually, it's caused by a missing VirtualBox template or a GitLab connectivity issue.

About CyberForge

CyberForge is an orchestration platform designed to provision dynamic cybersecurity challenge environments on-demand.

Operating primarily via VirtualBox, CyberForge provides high-fidelity simulations spanning everything from isolated OWASP Web Vulnerabilities to expansive Active Directory killchains.

Architecture

  • API Runtime: FastAPI & Python 3.11
  • Database: SQLAlchemy (PostgreSQL / SQLite)
  • Frontend: Vanilla JS + High Contrast "Mono" Design System
  • Provisioning: VBoxManage & CALDERA TTPs
